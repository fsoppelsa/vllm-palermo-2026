Framing Notes — Naive vs vLLM Comparison
=========================================

CONTEXT
-------
The naive baseline (01_naive_inference.py) runs on CPU because standard
PyTorch/transformers cannot load the model onto the Jetson GPU: the default
CUDA caching allocator issues a single ~1 GB contiguous cudaMalloc for the
embedding layer, which Jetson's Tegra IOMMU (NvMap) rejects with ENOMEM.
vLLM ships its own CUDA memory manager that handles unified-memory platforms
correctly. Standard transformers does not.

HOW TO PRESENT IT
-----------------
"I'm going to run the same model, same prompt, using vanilla HuggingFace
transformers. Before we even get to inference speed, watch what happens when
it tries to load the model."

  [script prints:]
  [!] CUDA load failed: RuntimeError
  [!] Jetson's Tegra IOMMU rejects PyTorch's large cudaMalloc requests.
  [!] This is exactly the problem vLLM solves — falling back to CPU.

"PyTorch's default memory allocator issues a single ~1 GB contiguous memory
request for the embedding layer. Jetson's IOMMU says no. Standard transformers
can't even get the model onto the GPU. vLLM ships its own CUDA memory manager
— it handles unified memory, fragmented allocation, and hardware quirks that
upstream PyTorch doesn't. That's not a workaround; that's production software."

Then run Part 2 (02_vllm_single.py), which loads and serves at full GPU speed.

WHY THE COMPARISON IS STILL FAIR
---------------------------------
The question being answered is NOT "CPU inference vs GPU inference."
It is "what you get with `pip install transformers` vs what you get with a
proper inference server." That framing is honest and more relevant to the
audience — most of them won't be tuning CUDA allocators, they'll be choosing
tools.

The comparison table in script 02 explicitly labels "naive (CPU)" vs
"vllm (GPU)" so the audience sees the difference honestly, not hidden.

KILLER RESPONSE TO PUSHBACK
----------------------------
"Even on x86 with a discrete GPU where HuggingFace CAN load to CUDA, vLLM
is still 5–10× faster at throughput — because continuous batching,
PagedAttention, and CUDA graphs are the real gains. The GPU access issue
here just makes the gap visible earlier and more dramatically."

SUMMARY
-------
This is actually a stronger demo point than GPU vs GPU would be.
The inability of standard transformers to even reach the GPU on constrained
unified-memory hardware is direct evidence of why production inference
infrastructure matters.
